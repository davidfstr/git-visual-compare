I have used the GitX program for visualizing diffs for over a decade,
using its command line interface: `gitx --diff sha1 sha2` to summon
a UI window displaying the unified diff generated by `git`.

However it is becoming increasingly painful to use `gitx` for this
purpose:
- It has no dark mode.
- There is no way to search (Find) within a displayed diff
- `gitx` is no longer maintained, and none of its forks have
  gained enough prominence to become the leading fork,
  as far as I can tell

I do end up using several somewhat advanced features of the
`gitx --diff ...` command-line support:
- `-w`: Ignore whitespace
- `sha1 sha2 -- file1 file2`: Limit to specific files
- `-U20`: Control the number of lines of context displayed

I have tried to find any existing tool that satisfies the above
requirements, including commercial tools. GUI tools I looked closely at:
- Kaleidoscope v6
    - ❌ Very expensive subscription pricing: $8-14/month
        - I want the same kind of unified diff style that I got for free
          10 years ago. I have no interest in supporting ongoing development
          for such a basic tool.
    - 💬 Integrates with CLI via the `git difftool -d` mechanism
- Beyond Compare
    - ❌ No unified diffs; only side-by-side
    - 💬 Integrates with CLI via the `git difftool -d` mechanism

The closest match for what I want seem to be tools with a TUI
(rather than a GUI). However TUI tools don't mesh well with my workflows
because my terminals are generally wide and short (e.g. 80 x 20), optimized
for launching programs and reading limited output, rather than being
very tall and wide, amendable for reading large multi-file diffs.

So - as a last resort - I am looking at building my own tool that satisfies
my criteria.

Recommended technical requirements:
- Implement in **Python**, a language I am extremely familiar with,
  with broad platform support (macOS/Windows/Linux)
- Implement UI via an integrated browser, Electron-style,
  but using the platform-native UI browser view, via **pywebview**
  <https://pywebview.flowrl.com/>
- Use the locally installed **git** tool to actually perform diffs.
  Strive to support the full `git diff <anything>` syntax,
  via a syntax like `gvc <anything>` (`gvc` = "Git Visual Compare",
  the proposed name of this new tool)
- Package/distribute as a macOS .app using **PyInstaller**
    - It would be nice to package as an .exe on Windows,
      or as a self-contained binary (?) on Linux,
      but that is less of a priority than initial macOS support
- Must support both **light & dark modes**.
    - Detect *immediately* if the operating system switches modes
      (because a sunset/sunrise transition has occurred and the mode
      is set to be auto/time-based), without needing to relaunch the
      program or reopen existing diff windows.
- Should display diffs in a fashion similar to `gitx --diff`:
    - Each file gets its own section, collapsible
    - Added lines use background color A; removed lines use background color B
        - No need for syntax highlighting
    - Added lines with trailing whitespace have the trailing whitespace
      highlighted with background color C
    - Seam lines that occur between adjacent hunks,
      with content like `@@ -3026,63 +3102,96 @@ class Resource:`
      highlighted with background color D
    - Prefer GitX's original colors when in light mode
    - Exception: At the top of the diff window display an outline
      of all files added/edited/removed/etc in the diff. Clicking
      a file warps you to the file's section in the diff.
- Must support **find** to locate substrings in displayed diffs.
    - Find... ⌘F
    - Find Next ⌘G
    - Find Previous ⇧⌘G

Future extensions:
- Automatic updates via appcast, Sparkle-style
    - IIRC PyInstaller has built-in support for this style of app updating
- A nice About Box
- E2E automated tests with **Playwright**,
  with test names in given-when-then (or full-sentence) format
- `git difftool -d` integration support, for potentially better integration
  with git graph visualizer tools like Fork
